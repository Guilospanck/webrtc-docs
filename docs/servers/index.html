Servers

In WebRTC we (might) have these servers.

Signaling server

Responsible for exchanging peers capabilities (SDP - Session Description Protocol) and ICE (Internet Connection Establishment) candidates. It’s only needed when a (re)negotiation is needed.

STUN/TURN server

These are used to gather ICE candidates. Usually the STUN server is gonna be a Google one (stun:stun.l.google.com:19302) and the TURN server will be either a self-hosted (like Coturn - TURN server ) or a 3rd party like Cloudflare TURN service or Twilio Network Traversal Service.

SFU (Selective Forwarding Unit) server

This will be necessary if you want to have group calls, as the P2P (mesh) topology doesn’t work well when more participants start to gather around (bandwidth starts to explode, clients have to handle many encodes/decodes).
